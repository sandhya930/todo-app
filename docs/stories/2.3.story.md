# Story 2.3: Projects

## Status: Draft

## Story

- As Sam (student) and Alex (professional)
- I want to organize tasks into named projects with a color and icon
- so that I can view all tasks for a course, client, or life area in one dedicated space

## Acceptance Criteria (ACs)

1. Users can create a project with: name (required, max 50 chars), color (12 preset options), and icon (emoji picker)
2. Tasks can be assigned to a project during capture or via edit after creation
3. Project view shows all tasks in that project grouped by status: Active, Upcoming, Someday, Completed (collapsible groups)
4. The navigation sidebar / tab shows each project with its color, icon, and a task-count badge (active tasks only)
5. Projects can be archived: archived projects disappear from the sidebar but their tasks remain searchable and accessible via a "Archived Projects" section
6. A default "Personal" project is pre-created for every new user; it cannot be deleted (only renamed)
7. Free tier: maximum 3 active (non-archived) projects; Pro tier: unlimited. Attempting to create a 4th project on free tier shows an upgrade prompt
8. Tasks not assigned to any project are displayed under the "Personal" project by default (not left orphaned)

## Tasks / Subtasks

- [ ] **Task 1 — Project data model** (AC: 1, 5, 6, 8)
  - [ ] 1.1 Define `Project` schema: `id` (UUID), `user_id`, `name` (string, max 50), `color` (enum of 12 color keys), `icon` (emoji string, max 2 chars), `is_default` (boolean), `archived_at` (nullable timestamp), `created_at`, `sort_order` (integer)
  - [ ] 1.2 Add `project_id` (nullable FK → Project) to Task model; default null maps to "Personal" project at query time
  - [ ] 1.3 Seed "Personal" project on user account creation with `is_default = true`
  - [ ] 1.4 Write unit tests: project creation, default project seeded on user creation, `is_default` project cannot be deleted (enforce in service layer)

- [ ] **Task 2 — Create / edit project UI** (AC: 1, 7)
  - [ ] 2.1 "New Project" button in sidebar/navigation; opens a bottom-sheet or modal form
  - [ ] 2.2 Form fields: text input for name (max 50, inline character count), color picker (12 swatches, single-select), emoji picker (system emoji keyboard or curated subset)
  - [ ] 2.3 Save creates project; Cancel dismisses without changes
  - [ ] 2.4 Edit project: accessible via long-press or kebab menu on project in sidebar; same form pre-populated
  - [ ] 2.5 Free tier enforcement: if user has 3 active projects and taps "New Project", show upgrade modal ("Upgrade to Pro for unlimited projects") instead of the creation form
  - [ ] 2.6 Write component tests: form validation (empty name blocked), color selection, emoji selection, free-tier gate shows upgrade modal at limit

- [ ] **Task 3 — Assign task to project** (AC: 2)
  - [ ] 3.1 During capture (QuickCaptureInput, Story 1.1): add an optional project selector below the title input (collapsed by default; tap "+" or project icon to expand)
  - [ ] 3.2 In task detail / edit view: project selector dropdown/picker shows all active projects + "No Project" option (which maps to Personal)
  - [ ] 3.3 Changing a task's project updates `project_id` in local DB and syncs
  - [ ] 3.4 Write tests: assign during capture, reassign from edit, unassign (falls back to Personal)

- [ ] **Task 4 — Project view screen** (AC: 3, 4)
  - [ ] 4.1 Create `ProjectView` screen: header shows project icon, name, color accent, task count; body shows tasks grouped by status
  - [ ] 4.2 Status groups: "Active" (status = active or inbox), "Upcoming" (due_date > today, not yet active), "Someday" (pinned to someday), "Completed" (last 30 days, older hidden behind "Show older")
  - [ ] 4.3 Each group is collapsible; collapsed state persisted per project per user
  - [ ] 4.4 Quick task add within ProjectView: captures task and auto-assigns to this project
  - [ ] 4.5 Write component tests: groups render correctly, collapsible toggle, quick add assigns correct project_id

- [ ] **Task 5 — Sidebar/navigation integration** (AC: 4, 5, 6)
  - [ ] 5.1 Sidebar (web) / nav menu (mobile): list all active projects sorted by `sort_order`, each showing icon + name + active-task count badge
  - [ ] 5.2 "Personal" project always first in list (pinned, not draggable)
  - [ ] 5.3 Archive project: right-click / kebab → "Archive" → confirm dialog → project removed from sidebar; tasks remain
  - [ ] 5.4 "Archived Projects" collapsible section at bottom of sidebar shows archived projects; tap to view (read-only, no new tasks)
  - [ ] 5.5 Unarchive: button in archived project view restores it to active list
  - [ ] 5.6 Write tests: sidebar lists active projects with badge; archive removes from active list; unarchive restores; Personal always first

- [ ] **Task 6 — Orphan task migration** (AC: 8)
  - [ ] 6.1 Any task with `project_id = null` is displayed as belonging to "Personal" in all views (query-time join, not a data migration)
  - [ ] 6.2 When displaying "Personal" project task count badge, include tasks with `project_id = null`
  - [ ] 6.3 Write unit test: task with null project_id appears in Personal project view

## Dev Notes

### Dependencies
- Task model from EP-01 Story 1.1 (must exist before extending with `project_id`)
- Today view from Story 2.1 (project color accent should be visible on task cards in Today — minor visual enhancement, not blocking)
- Quick capture from Story 1.1 (project selector is an extension of that component)

### Free vs Pro Tier Enforcement (from PRD §10 Phase 1)
- Free tier: 3 active projects max — enforced server-side (not client-only) to prevent circumvention via API
- "Active" = `archived_at IS NULL`; archived projects do not count toward the limit
- The "Personal" default project counts as 1 of the 3 free projects
- Upgrade prompt must be non-blocking: user can dismiss and continue using the app with existing projects

### Color System
- 12 preset colors (not free-form hex) for visual consistency
- Suggested palette (adjust with design): Red, Orange, Amber, Yellow, Lime, Green, Teal, Cyan, Blue, Indigo, Purple, Pink
- Store as a color key string (e.g., `"blue"`) not a hex value — hex values are resolved in the UI layer from a theme map

### Emoji Picker
- On mobile: use the system emoji keyboard (simplest implementation)
- On web: use a lightweight emoji picker library (e.g., emoji-picker-element) or limit to a curated set of ~50 productivity-relevant emojis
- Store as a single emoji character (Unicode); max 2 chars to handle multi-codepoint emoji (e.g., 🏠)

### "Completed" Group in Project View
- Only show last 30 days of completed tasks by default — long-completed tasks create noise
- "Show older completed tasks" link loads all completed tasks for that project (paginated, 50 at a time)

### What This Story Does NOT Include
- Shared/collaborative projects (EP-05 Story 5.2)
- Project-level default energy level (EP-02 Story 2.4 — done separately)
- Sub-projects or project nesting
- Project templates (Phase 3 platform)
- Project-level sorting preferences beyond manual sort_order
