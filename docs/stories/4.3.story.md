# Story 4.3: Weekly Accomplishment Digest

## Status: Draft

## Story

- As Alex (overwhelmed professional)
- I want a weekly email showing what I accomplished this week
- so that I have a tangible record of progress to reference in performance reviews and optionally share with others

## Acceptance Criteria (ACs)

1. A digest email is sent every Sunday evening (default 7pm user's local time; day and time configurable in Settings)
2. The digest is **opt-in only**: the user is prompted to enable it during onboarding; it can be enabled/disabled in Settings → Notifications → Weekly Digest
3. The email contains: total tasks completed this week, top project by completion count, current streak, personal record comparison ("Your best week: N tasks"), and a highlighted "star task" (longest-standing completed task of the week)
4. A one-tap unsubscribe link in the email footer complies with CAN-SPAM and GDPR; clicking it disables the digest without requiring login
5. An in-app version of the digest is available in Profile → Weekly Reports, showing the last 12 weeks of summaries
6. Tapping "Share" in the in-app digest generates a shareable PNG card with the week's stats for posting to social media
7. The digest email renders correctly in: Gmail (web + mobile), Apple Mail (iOS), Outlook (web); tested via a rendering tool or snapshot
8. If a user completed 0 tasks in a week, no digest email is sent for that week (silence is better than a reminder of non-productivity)

## Tasks / Subtasks

- [ ] **Task 1 — Digest preferences on User model** (AC: 1, 2)
  - [ ] 1.1 Add to User settings: `digest_enabled` (boolean, default false), `digest_day` (0=Sun–6=Sat, default 0), `digest_hour` (0–23, default 19), `digest_unsubscribe_token` (UUID, generated on account creation, never changes)
  - [ ] 1.2 Onboarding flow: after account creation, show a prompt "Get a weekly wins email every Sunday?" with "Yes, sign me up" / "Maybe later" buttons; "Yes" sets `digest_enabled = true`
  - [ ] 1.3 Settings → Notifications → Weekly Digest: toggle on/off, day picker, time picker
  - [ ] 1.4 Write unit tests: default state (disabled), opt-in sets flag, settings persisted

- [ ] **Task 2 — Digest data aggregation** (AC: 3, 8)
  - [ ] 2.1 Create `DigestAggregator.buildWeeklySummary(userId, weekStartDate)`:
    - `tasks_completed_count`: count of tasks with `completed_at` in the week window (Mon 00:00 – Sun 23:59 user timezone)
    - `top_project`: project with most completed tasks this week (name + color + count); null if no projects used
    - `current_streak`: from `StreakService` (Story 4.2)
    - `personal_best_week_count`: max `tasks_completed_count` across all previous weeks for this user (from `DailyCompletion` table)
    - `star_task`: task completed this week with the oldest `created_at` (longest-standing commitment fulfilled)
  - [ ] 2.2 If `tasks_completed_count = 0`: return null (no digest sent — AC 8)
  - [ ] 2.3 Write unit tests: correct task count for week window; top project tiebreak (alphabetical); star task selection (oldest created_at); null returned for zero completions; personal best calculation

- [ ] **Task 3 — Email template** (AC: 3, 7)
  - [ ] 3.1 Build HTML email template using table-based layout (required for Outlook compatibility — no flexbox/grid in email)
  - [ ] 3.2 Template sections:
    - Header: app logo + "Your week in review"
    - Hero stat: large "You completed N tasks this week 🎉"
    - Stats row: streak | top project | personal best
    - Star task highlight: "This week's highlight: [task title]"
    - CTA: "Open app" button
    - Footer: unsubscribe link, app name, address (CAN-SPAM requirement)
  - [ ] 3.3 Subject line: "🔥 You crushed N tasks this week!" (personalised with count)
  - [ ] 3.4 All colours use inline styles; no external CSS (email client compatibility)
  - [ ] 3.5 Responsive: single-column below 600px width
  - [ ] 3.6 Write snapshot test: rendered HTML contains all required sections; unsubscribe link present with correct token; subject line correct

- [ ] **Task 4 — Email send job** (AC: 1, 4, 8)
  - [ ] 4.1 Server-side scheduled job: runs Sunday at each user's configured `digest_hour` (timezone-aware scheduling, same pattern as deferral job)
  - [ ] 4.2 Job queries users where: `digest_enabled = true` AND `tasks_completed_count > 0` this week
  - [ ] 4.3 Sends email via the configured email provider (SendGrid / SES / Postmark — TBD in architecture; abstract behind `EmailSendService`)
  - [ ] 4.4 One-click unsubscribe: `GET /api/v1/unsubscribe?token={digest_unsubscribe_token}` sets `digest_enabled = false` — no auth required, token-only; returns a plain HTML confirmation page "You've been unsubscribed. [Re-enable in settings]"
  - [ ] 4.5 Unsubscribe endpoint must work without the user being logged in (CAN-SPAM requirement)
  - [ ] 4.6 Write unit tests: job sends to enabled users with completions; skips users with 0 completions; skips disabled users; unsubscribe endpoint sets flag, no auth needed

- [ ] **Task 5 — In-app Weekly Reports** (AC: 5)
  - [ ] 5.1 Profile → Weekly Reports screen: list of past 12 weeks, each showing: date range, tasks completed, streak at week end, top project
  - [ ] 5.2 Tap any week → expand to show full digest summary (same data as email)
  - [ ] 5.3 "Share" button on each week summary (see Task 6)
  - [ ] 5.4 Empty state for weeks with 0 completions: show week in list but greyed out with "No completions this week" (do not hide — absence is informative)
  - [ ] 5.5 Write component tests: 12 weeks listed; zero-completion week shown greyed; expand/collapse works

- [ ] **Task 6 — Shareable PNG card** (AC: 6)
  - [ ] 6.1 "Share" tap → generate a PNG image (400×400px) using canvas rendering:
    - App branding (name + icon)
    - "I completed N tasks this week 🎉"
    - Streak: 🔥 N days
    - Top project name + colour swatch
    - Week date range
  - [ ] 6.2 On mobile: invoke native share sheet (`Share.share()` in RN) with the PNG
  - [ ] 6.3 On web: trigger PNG download via `<a download>` link
  - [ ] 6.4 Write unit tests: PNG generated with correct dimensions; share sheet invoked on mobile; download triggered on web

## Dev Notes

### Dependencies
- `DailyCompletion` table and `StreakService` (Story 4.2) — digest aggregator reads from these
- Task `completed_at` field — add if not already on Task model (Story 1.1 may not have included it; add now)
- User `timezone` field (Story 3.2) — required for digest send timing and week boundary calculation
- Midnight job scheduler (Story 3.2) — extend or create a parallel job for digest sends

### `completed_at` Field
If `completed_at` is not yet on the Task model (EP-01 Story 1.1 only defined `created_at`), add it now:
- `completed_at`: nullable timestamp; set by `completeTask()` service; never changed after set
- Needed for: digest aggregation, weekly reports, potential future analytics

### Email Client Compatibility (AC 7)
- **Outlook** does not support CSS flexbox, grid, or `<style>` tags — use `<table>` layout with inline styles only
- **Gmail** clips emails > 102KB — keep template under 50KB
- **Dark mode:** add `@media (prefers-color-scheme: dark)` in a `<style>` tag for clients that support it (Gmail on iOS, Apple Mail); Outlook will ignore it
- Test tool recommendation: Litmus or Email on Acid (or free alternative: send to real accounts on each client)

### CAN-SPAM / GDPR Compliance (AC 4)
Required in every marketing email:
1. Sender name and physical address in footer
2. Clear "Unsubscribe" link that works without login
3. Honour unsubscribe within 10 business days (our token approach is instant — better than required)
4. Subject line must not be misleading

GDPR additional: user gave explicit opt-in (onboarding prompt), record stored with timestamp in `digest_opted_in_at` field

### Shareable Card (AC 6)
- Use `html-to-image` or `react-native-view-shot` for in-app PNG generation
- Card must look good on both light and dark social media feeds — use a solid brand-coloured background
- Do not include the user's full name or email on the shareable card (privacy)

### What This Story Does NOT Include
- Digest for other time periods (daily, monthly) — weekly only for MVP
- HTML → plain text fallback email (nice-to-have; defer to Phase 2)
- Push notification version of the digest
- Third-party calendar summary (e.g., "compared to last week you completed 20% more tasks" phrasing — Phase 2)
