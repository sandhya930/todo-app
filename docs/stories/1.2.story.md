# Story 1.2: Voice Task Capture

## Status: Review

## Story

- As Alex (overwhelmed professional)
- I want to dictate a task by voice
- so that I can capture ideas while commuting, cooking, or otherwise hands-free without breaking my flow

## Acceptance Criteria (ACs)

1. A dedicated microphone button is accessible on the home screen without navigating away
2. Tapping the mic button starts on-device voice transcription (no audio sent to server without explicit user opt-in)
3. Natural language date/time phrases in the dictation are extracted and set as `due_date` (e.g., "remind me tomorrow at 9am" â†’ due date set accordingly)
4. The user sees the live transcription text before confirming save; they can edit the text inline before saving
5. Voice capture is fully available offline (on-device model; no network required)
6. If microphone permission has not been granted, a clear explanation is shown with a link to device Settings â€” voice capture does not silently fail
7. A "Try again" button is available if transcription produces an empty or garbled result
8. Confirmed task is saved via the same `createTask()` path as Story 1.1 and appears in Inbox instantly

### On-Device STT Platform Notes
- **iOS:** `SFSpeechRecognizer` supports on-device mode since iOS 13 (`requiresOnDeviceRecognition = true`)
- **Android:** `SpeechRecognizer` can operate offline with downloaded language pack; check availability before starting
- **Web:** `Web Speech API` is browser-native but NOT guaranteed offline; if offline, show graceful fallback ("Voice capture requires an internet connection on web") â€” web voice is P2 polish, not MVP blocker
- Audio is never stored to disk or sent to server in any flow

### Privacy (from PRD Â§7.3)
- Task content privacy is explicitly called out in PRD â€” on-device processing is non-negotiable for voice
- A privacy notice ("Voice is processed on your device only") should appear the first time the user taps the mic button

### Performance (from PRD Â§7.1)
- Transcription should begin within 500ms of mic button tap
- Review screen should appear within 1 second of user stopping dictation

### Accessibility
- Mic button must have `aria-label="Start voice capture"` and equivalent accessibility labels on iOS/Android
- Recording state must be communicated to screen readers (not just visual pulse animation)
- All review screen actions keyboard-accessible on web

### What This Story Does NOT Include
- Background listening / always-on voice ("Hey Todo")
- Multi-language transcription (English only at launch)
- Audio attachment stored with task â€” only the text transcript is saved

### Completion Notes List

- **Web Speech API is NOT fully offline (AC 5):** On Chromium-based browsers the Web Speech API routes audio to Google's servers. True on-device voice (iOS `SFSpeechRecognizer`, Android `SpeechRecognizer`) is deferred to the mobile pass (React Native/Expo not yet set up). A first-use privacy notice informs web users of this limitation.
- **NLP date extraction stubbed (AC 3):** `extractDateFromText()` in `packages/web/src/lib/date-extraction.ts` returns `{ cleanTitle: text, dueDate: null }` until Story 1.5 wires in Chrono.js. The review screen UI for the due-date chip is fully implemented.
- **Mobile (iOS/Android) deferred:** `VoiceCaptureButton` renders `null` when `isVoiceCaptureSupported()` returns false (e.g. React Native environment). Mobile STT requires the Expo package setup planned for the mobile pass.
- **`PRIVACY_NOTICE_KEY` in localStorage:** First-use privacy notice flag (`voice_privacy_notice_seen`) is stored in `localStorage`. Tests clear this in `beforeEach`/`afterEach`.
