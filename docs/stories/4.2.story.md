# Story 4.2: Streak Counter

## Status: Draft

## Story

- As Jordan (ADHD user)
- I want a streak counter that tracks my consecutive days of completing at least one task
- so that I'm motivated to maintain the daily habit even on difficult days

## Acceptance Criteria (ACs)

1. The streak counter is visible in the Today view header: flame icon 🔥 + current streak count (e.g., "🔥 7")
2. The streak increments when at least 1 task is completed in a calendar day (user's local timezone)
3. The streak resets to 0 if 0 tasks are completed by midnight of any day (after the deferral job runs — streak check happens after deferral, not before)
4. Grace day mechanic: once per calendar month the user can tap the streak counter and activate a "grace day" which preserves the streak on a zero-completion day; grace day balance (0 or 1) is shown in the streak detail view
5. Milestone notifications (in-app + optional push) at streak counts: 3, 7, 14, 30, 60, 100 days
6. A streak history is accessible from the streak counter: a calendar heatmap showing daily completion counts for the last 90 days (GitHub contribution graph style)
7. Completing a deferred task counts normally toward that day's streak — no penalty for deferral history
8. Completing the weekly review (EP-03 Story 3.3 `WEEKLY_REVIEW_COMPLETED` event) counts as a completion for streak purposes on that day
9. The streak display in Today view header was stubbed as "0" in EP-02 Story 2.1 — this story replaces that stub with the real implementation

  - [ ] 1.1 Add to User model: `current_streak` (integer, default 0), `longest_streak` (integer, default 0), `last_completion_date` (nullable date string `YYYY-MM-DD`), `grace_days_used_this_month` (integer, default 0), `grace_day_reset_month` (string `YYYY-MM`, for monthly reset tracking)
  - [ ] 1.2 Add `DailyCompletion` table: `user_id`, `date` (YYYY-MM-DD), `completion_count` (integer) — upserted each time a task is completed; drives the heatmap
  - [ ] 1.3 Write unit tests: fields initialise correctly; `DailyCompletion` upsert increments count correctly for same-day multiple completions

- [ ] **Task 2 — Streak calculation service** (AC: 2, 3, 7, 8)
  - [ ] 2.1 Create `StreakService` with:
    - `recordCompletion(userId, date)`: upsert `DailyCompletion` for date; if `last_completion_date != date`, increment `current_streak`, update `last_completion_date`; if `current_streak > longest_streak`, update `longest_streak`
    - `checkAndResetStreak(userId, date)`: called by midnight job after deferral job; if `last_completion_date < yesterday` AND no grace day active → reset `current_streak = 0`
    - `activateGraceDay(userId, date)`: if `grace_days_used_this_month < 1` → set `last_completion_date = date`, increment grace_days_used_this_month; else return error "No grace days remaining this month"
  - [ ] 2.2 Monthly grace day reset: at start of each calendar month, reset `grace_days_used_this_month = 0` and update `grace_day_reset_month`
  - [ ] 2.3 `recordCompletion` is called by: `completeTask()` (any task completion), `WEEKLY_REVIEW_COMPLETED` event handler
  - [ ] 2.4 Write unit tests: first completion of day increments streak; second completion same day no double-increment; missed day resets streak; grace day preserves streak; grace day exhausted returns error; weekly review completion counts

- [ ] **Task 3 — Streak display in Today header** (AC: 1, 9)
  - [ ] 3.1 Replace the stubbed `streak={0}` in Today view header (EP-02 Story 2.1) with live data from `StreakService`
  - [ ] 3.2 Streak badge: 🔥 icon + number; tapping opens the streak detail sheet (Task 4)
  - [ ] 3.3 Animate streak count increment: brief scale-up (1.0 → 1.3 → 1.0, 300ms) when count increases during the current session
  - [ ] 3.4 Write component tests: correct count displayed; tap opens detail sheet; increment animation fires on count change

- [ ] **Task 4 — Streak detail sheet + grace day** (AC: 4, 6)
  - [ ] 4.1 Bottom sheet or modal showing: current streak, longest streak, grace day balance ("1 grace day available this month" or "No grace days remaining"), calendar heatmap, "Use grace day" button (disabled if balance = 0 or today already has a completion)
  - [ ] 4.2 Calendar heatmap: 90-day grid, 7 columns (Mon–Sun), cells coloured by completion count:

    - 1–2: light brand colour
    - 3–4: medium brand colour

  - [ ] 4.3 "Use grace day" tap: calls `StreakService.activateGraceDay(userId, today)`; updates UI immediately; shows "Grace day used — streak preserved ✓"; disables button for remainder of month
  - [ ] 4.4 Write component tests: heatmap renders 90 cells; grace day button disabled when balance=0; disabled when today already has completion; "Used" state shown after activation

  - [ ] 5.2 If milestone reached: dispatch in-app notification ("🔥 7-day streak! You're on fire.") with a celebratory icon

  - [ ] 5.4 Optional push notification for milestones: only sent if user has enabled push notifications (separate opt-in — default off for this story)
  - [ ] 5.5 Each milestone notification fires only once per streak (not re-triggered if user maintains beyond the milestone count)

## Dev Notes

### Dependencies
- `completeTask()` service (EP-01 1.1) — must call `StreakService.recordCompletion()` after successful completion
- Today view header stub (EP-02 2.1) — replace `streak={0}` with live query
- Midnight deferral job (EP-03 3.2) — extend to call streak reset after deferral pass
- `WEEKLY_REVIEW_COMPLETED` event (EP-03 3.3) — wire to `StreakService.recordCompletion()`
- `CompletionAnimationService` (Story 4.1) — called on milestone events

### Timezone Consistency
- All date strings stored as `YYYY-MM-DD` in the user's **local timezone**
- Use the same `timezone` field on User model introduced in Story 3.2
- Never compare dates in UTC when determining streak continuity — a user completing a task at 11:59pm local time must count for that local day

### Grace Day UX Notes (from PRD §4.4 Persona Jordan)
- Grace day is about emotional safety, not gaming the system
- The detail sheet copy should be warm: "Life happens. Your grace day keeps your streak intact for one day this month."
- The "No grace days remaining" state should be empathetic, not punitive: "You've used your grace day for March. Your next one resets in April."
- Do not show grace day prominently in the main Today header — it lives in the detail sheet only

### Heatmap Implementation
- Build as a custom component; no heavyweight charting library needed
- 90 days × 7 columns = 90 cells rendered as a flat list/grid
- Each cell is a small square (8×8px on mobile, 12×12px on web) with rounded corners
- Accessibility: cells have `aria-label="[date]: [N] tasks completed"` for screen readers

### Streak Reset vs. Deferred Completions (AC 3 + AC 7)
- The midnight job order must be: (1) run deferral job, (2) run streak reset check
- This ensures a task deferred to today (from yesterday) that the user then completes still counts today
- Completing a deferred task calls `recordCompletion(today)` → streak maintained

### What This Story Does NOT Include
- Streak leaderboard or social comparison
- Streak "insurance" (buying extra grace days — future monetisation idea)
- Per-project streaks
- Streak restore if a past date was actually completed (no retroactive edits)
