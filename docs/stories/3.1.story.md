# Story 3.1: AI "Pick for Me"

## Status: Draft

## Story

- As Alex (overwhelmed professional)
- I want an AI button that recommends the single best task to work on right now
- so that I can skip decision fatigue and start immediately without deliberating over my list

## Acceptance Criteria (ACs)

1. A "Pick for me" button is prominently displayed in the Today view and accessible via keyboard shortcut `P` (web/desktop)
2. The algorithm selects exactly one task and shows a one-sentence explanation of why it was chosen (e.g., "This is due today and matches your current Low Focus energy level")
3. The algorithm considers, in priority order: (a) P0-today deadline tasks, (b) energy level match to user's declared current state, (c) estimated duration vs. time available until next calendar event (stub: time available = remaining hours in workday), (d) historical completion patterns (stub: skip in v1)
4. The user can tap "Not this one" to skip the suggestion and receive a different one; maximum 3 skips per session
5. After 3 skips, the button shows: "That's all my suggestions for now — check your Inbox for more tasks"
6. If no suitable tasks exist (Today is empty, all tasks are Someday/Completed): friendly message "Your slate is clear — add something or check Someday", no error state
7. The recommendation is computed client-side (rule-based v1); no network call required; result available within 200ms of button tap
8. Tapping the recommended task card opens it directly in Focus Mode (Story 2.2)

## Tasks / Subtasks

- [ ] **Task 1 — Pick-for-me algorithm service (v1 rule-based)** (AC: 3, 7)
  - [ ] 1.1 Create `pickForMe(tasks: Task[], userState: UserState): PickResult` pure function
    - Input: array of today's active tasks, `userState = { currentEnergyLevel, workdayEndsAt }`
    - Output: `{ task: Task, reason: string } | null`
  - [ ] 1.2 Scoring logic (apply in order, first match wins):
    1. Hard due today + overdue-deferred count ≥ 2 → highest urgency
    2. Due today (not yet deferred) → high urgency; sort by due time ascending
    3. Energy level matches `userState.currentEnergyLevel` (if set) → preference boost
    4. `estimated_duration_minutes` ≤ remaining workday minutes → fits in remaining time
    5. Fallback: first task in `today_sort_order`
  - [ ] 1.3 Generate reason string from whichever rule triggered (see reason templates below)
  - [ ] 1.4 Exclude from candidates: tasks with `status = 'someday'`, `status = 'completed'`, `status = 'archived'`; tasks already skipped in current session
  - [ ] 1.5 Write unit tests: due-today task selected first; energy match boosts; duration fit considered; skip list exclusion; empty input returns null

- [ ] **Task 2 — User current energy state** (AC: 2, 3)
  - [ ] 2.1 Add a lightweight "current energy" selector to the Today view header: small segmented control (🔥 / ⚡ / 🌿) + "Not set" default
  - [ ] 2.2 State is session-only (not persisted across app restarts in v1)
  - [ ] 2.3 "Pick for me" reads `currentEnergyLevel` from this selector when computing recommendation
  - [ ] 2.4 If `currentEnergyLevel` is not set, energy matching step is skipped (algorithm still runs on other criteria)
  - [ ] 2.5 Write component tests: selector renders, selection updates state, "Not set" is default, value passed to pickForMe

- [ ] **Task 3 — "Pick for me" button and result UI** (AC: 1, 2, 4, 5, 6, 8)
  - [ ] 3.1 Add "Pick for me" button to Today view: prominent placement (below header, above task list); icon: dice or sparkle ✨; label: "Pick for me"
  - [ ] 3.2 Keyboard shortcut `P` (web): triggers Pick for me when Today view is active and no input is focused
  - [ ] 3.3 On tap: run `pickForMe()` (synchronous, client-side); animate button briefly (200ms spinner or shimmer)
  - [ ] 3.4 Result card: slides in below the button; shows task title (large), reason string (small, muted), "Start in Focus Mode" button, "Not this one →" link
  - [ ] 3.5 "Start in Focus Mode": calls `enterFocusMode(task)` (Story 2.2); dismisses result card
  - [ ] 3.6 "Not this one": adds task to session skip list; re-runs `pickForMe()`; updates result card; counter tracks skips
  - [ ] 3.7 After 3 skips: replace result card with "That's all my suggestions for now…" message with "See all tasks" link to Inbox
  - [ ] 3.8 Empty result (null from pickForMe): show "Your slate is clear…" message (AC 6)
  - [ ] 3.9 Write component tests: result card renders with task + reason; "Not this one" cycles through suggestions; 3-skip limit shows final message; empty result shows clear-slate message

- [ ] **Task 4 — Reason string templates** (AC: 2)
  - [ ] 4.1 Define reason templates for each scoring rule:
    - Due today + high deferral count: "This has been deferred {N} times — today's the day."
    - Due today: "Due today and ready to go."
    - Energy match + due today: "Due today and matches your {energy} energy level."
    - Energy match only: "Matches your current {energy} energy — a good fit right now."
    - Duration fit: "Can be done in ~{N} min, which fits before your day ends."
    - Fallback: "Your top task for today."
  - [ ] 4.2 Reason string must be a single sentence, max 60 characters; truncate gracefully if dynamic values overflow
  - [ ] 4.3 Write unit tests: each rule generates the correct reason string; dynamic values interpolated correctly

- [ ] **Task 5 — Analytics stub** (AC: 7)
  - [ ] 5.1 Log a local analytics event on each "Pick for me" interaction: `{ event: 'pick_for_me_used', taskId, ruleApplied, skipsBeforeAccept, accepted: bool }`
  - [ ] 5.2 Events stored locally for now (full analytics pipeline is out of scope); this instrumentation enables future ML model training
  - [ ] 5.3 Write unit test: event logged with correct fields on accept and on skip

## Dev Notes

### Dependencies
- Task model with `energy_level`, `estimated_duration_minutes`, `today_sort_order`, `status` (EP-01 Story 1.1, EP-02 Stories 2.1, 2.4)
- Focus Mode entry point (EP-02 Story 2.2) — "Start in Focus Mode" calls `enterFocusMode(task)`
- Today view (EP-02 Story 2.1) — button lives in this view
- Deferral count: requires `deferred_count` field on Task (added in Story 3.2); stub with 0 if 3.2 not yet done

### v1 vs v2 Algorithm
- **v1 (this story):** Pure rule-based, client-side, no network, deterministic
- **v2 (Phase 2):** ML model trained on user's historical `pick_for_me_used` events; cloud-assisted; requires opt-in
- The `pickForMe()` function interface must remain stable between v1 and v2 — v2 replaces the implementation, not the signature

### "Time Available" Stub (AC 3c)
- In v1: `timeAvailableMinutes = minutesUntilEndOfWorkday(now(), workdayEndsAt: '17:00')`
- `workdayEndsAt` defaults to 17:00 local time; user-configurable in a future story
- If current time > workdayEndsAt: `timeAvailableMinutes = 60` (assume at least 1 hour available)
- Calendar integration (reading real next event) is Phase 2

### Session Skip List
- Stored in component state only (React state / local variable) — not persisted to DB
- Cleared on: app backgrounded + foregrounded, navigating away from Today and returning
- Max 3 skips enforced by the UI layer (not the algorithm) — `pickForMe()` itself has no skip concept

### What This Story Does NOT Include
- ML-based recommendation (v2, Phase 2)
- Calendar event awareness for "time available" (Phase 2)
- Historical completion patterns in scoring (v2)
- "Pick for me" in views other than Today
- Persistent energy level setting (session-only in v1)
