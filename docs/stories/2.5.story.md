# Story 2.5: Someday List

## Status: Draft

## Story

- As Alex (overwhelmed professional)
- I want a "Someday" list that stores ideas without a committed date
- so that I can capture everything without pressure, knowing low-priority ideas won't clutter my Today view

## Acceptance Criteria (ACs)

1. A "Someday" list is accessible from the main navigation (sidebar on web, tab/menu on mobile)
2. Tasks in Someday have no due date and are excluded from the Today view, Upcoming view, and all overdue/deferral logic
3. Tasks can be quickly moved to Someday via: a swipe action on mobile (swipe-left reveals "Someday" action), a context menu action on web, and a "Someday" button in the quick-capture flow
4. The Someday list supports sorting by: Date Added (default), Title (alphabetical), Project, and Energy Level
5. The Weekly Review wizard in EP-03 Story 3.3 surfaces Someday items for triage â€” this story provides the data layer; EP-03 consumes it
6. Items that have not been interacted with (no edit, no view, no triage) for 90 days are automatically moved to an archived state, with a 7-day warning notification before archiving
7. Auto-archived Someday tasks are recoverable: accessible under a "Someday Archive" section; can be restored to active Someday at any time
8. Someday tasks are included in full-text search results

## Dev Notes

### Status Enum Decision (Task 1.1)
- **Recommended:** Add `'someday'` as a dedicated `status` value rather than a separate boolean field
- This keeps the Task model's status field as the single source of routing truth (consistent with `inbox`, `active`, `deferred`, `completed`, `archived`)
- Update the status enum in the Task model (EP-01 Story 1.1) â€” coordinate with dev if 1.1 is already in production
- **If adding a new status value is a breaking change to existing serialization**: use a migration + backfill; do not use a parallel boolean

### `last_interacted_at` Triggers
Update `last_interacted_at` on:
- Any field edit (title, notes, due_date, energy_level, project)
- Opening task detail view (even just reading it counts as interaction)
- Triage in weekly review
- Moving back from Someday to Inbox ("Activate")

Do NOT update on:
- Background sync writes
- Auto-archive warning notification delivery

### Exclusion from Overdue / Deferral Logic (AC 2)
- EP-03's midnight deferral job (Story 3.2) must explicitly skip tasks with `status = 'someday'`
- Someday tasks have no `due_date` by definition â€” but if a user somehow sets a due_date on a Someday task via API, the deferral job should still skip them

### Auto-Archive Notification Batching
- The 7-day warning is a single in-app notification for all qualifying tasks (not one per task)
- Count the qualifying tasks and show: "N Someday tasks will auto-archive in 7 days â€” [Review]"
- Do not send push notification for this; in-app notification only (notification center / badge)

### What This Story Does NOT Include
- Weekly Review wizard integration (EP-03 Story 3.3 consumes Someday data)
- Bulk operations on Someday (select all, bulk archive) â€” future enhancement
- Someday tasks visible in the Today view even as a "peek" option
- Sharing Someday tasks with others
