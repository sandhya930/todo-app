# Story 2.4: Energy Level Tags

## Status: Draft

## Story

- As Alex (overwhelmed professional)
- I want to tag tasks with an energy level (High Focus / Low Focus / No-Brainer)
- so that I can match tasks to my current cognitive state instead of forcing deep work when I'm tired or distracted

## Acceptance Criteria (ACs)

1. Energy level is an optional field on every task: `high_focus` | `low_focus` | `no_brainer` | `unset` (default)
2. Energy level can be set during task capture and in the task edit/detail view
3. Energy level is visible as a small colored icon/badge on task cards in all views (Today, Inbox, Projects, Someday); display is optional and can be toggled off in Settings
4. Users can filter any view (Today, Inbox, Project) by energy level; filter state persists per-view per-session
5. The "Pick for me" AI feature in EP-03 Story 3.1 factors energy level into its recommendation — this story only provides the data; EP-03 consumes it
6. A default energy level can be set per project (e.g., "Work" project defaults to `high_focus`); new tasks created within that project inherit the default unless overridden
7. Energy level icons are accessible: color is not the sole differentiator (icon shape also differs per level)

## Tasks / Subtasks

- [ ] **Task 1 — Energy level field on Task model** (AC: 1)
  - [ ] 1.1 Add `energy_level` field to Task schema: enum `'high_focus' | 'low_focus' | 'no_brainer' | null` (null = unset); default null
  - [ ] 1.2 Update `createTask()` and `updateTask()` services to accept and validate `energy_level`
  - [ ] 1.3 API (Story 1.4): `energy_level` already in the accepted fields per AC 1 of that story — verify it validates against the enum
  - [ ] 1.4 Write unit tests: valid enum values accepted, invalid value rejected, null accepted

- [ ] **Task 2 — Energy level selector UI** (AC: 2)
  - [ ] 2.1 Create `EnergyLevelPicker` component: three-option segmented control or icon row
    - 🔥 High Focus (red/orange) — deep work, requires full concentration
    - ⚡ Low Focus (yellow/amber) — routine work, some concentration needed
    - 🌿 No-Brainer (green) — automatic tasks, can be done on autopilot
    - (blank/grey) Unset — no energy tag
  - [ ] 2.2 Add `EnergyLevelPicker` to `QuickCaptureInput` (collapsed by default; expanded via a small energy icon tap)
  - [ ] 2.3 Add `EnergyLevelPicker` to task detail/edit view (always visible)
  - [ ] 2.4 Write component tests: each option selectable, selection updates state, unset option clears selection

- [ ] **Task 3 — Energy level badge on task cards** (AC: 3, 7)
  - [ ] 3.1 Add small energy level badge to `TaskCard` component: icon + color, positioned consistently (e.g., left of title or trailing end)
  - [ ] 3.2 Badge is hidden when `energy_level = null`
  - [ ] 3.3 Icon shapes (for accessibility — not color alone):
    - High Focus: 🔥 flame or up-arrow icon
    - Low Focus: ⚡ bolt or horizontal-bar icon
    - No-Brainer: 🌿 leaf or circle icon
  - [ ] 3.4 Settings → Display → "Show energy level badges": toggle on/off; preference stored per user; default on
  - [ ] 3.5 Write component tests: badge shown when energy_level set, hidden when null, correct icon per level, hidden when setting toggled off

- [ ] **Task 4 — Filter by energy level** (AC: 4)
  - [ ] 4.1 Add energy level filter control to Today, Inbox, and Project views: three toggle chips (🔥 / ⚡ / 🌿) above the task list; multiple selections allowed; "All" deselects all filters
  - [ ] 4.2 When a filter is active, only tasks matching the selected energy level(s) are shown; tasks with `energy_level = null` are hidden when any filter is active
  - [ ] 4.3 Filter state persists per-view for the current session only (not across app restarts)
  - [ ] 4.4 A visual indicator (e.g., filter icon turns colored) shows when a filter is active
  - [ ] 4.5 Write tests: apply high_focus filter → only high_focus tasks shown; combine two filters → both shown; clear filter → all tasks shown; null-energy tasks hidden when filter active

- [ ] **Task 5 — Project-level default energy** (AC: 6)
  - [ ] 5.1 Add `default_energy_level` field to Project schema: same enum as Task, default null
  - [ ] 5.2 In project edit form (Story 2.3): add `EnergyLevelPicker` for default energy level (optional)
  - [ ] 5.3 In `createTask()`: if task has `project_id` and `energy_level` is not explicitly set, inherit `project.default_energy_level`
  - [ ] 5.4 Inherited value is editable per-task (not locked); changing the project default does NOT retroactively update existing tasks
  - [ ] 5.5 Write unit tests: task created in project with default → inherits default; task with explicit energy_level set → default not applied; project default changed → existing tasks unchanged

## Dev Notes

### Dependencies
- Task model from EP-01 Story 1.1 (extend with `energy_level` field)
- `QuickCaptureInput` from Story 1.1 (extend UI)
- `TaskCard` component from Story 2.1 (extend with badge)
- Project model from Story 2.3 (extend with `default_energy_level`)
- EP-03 Story 3.1 "Pick for me" consumes `energy_level` — this story is the data provider

### Accessibility (from PRD §7.4 + AC 7)
- Energy badges must not rely on color alone — icon shape is the primary differentiator
- Badge must have `aria-label="Energy level: High Focus"` (or appropriate) on web
- On mobile: accessibilityLabel set for VoiceOver / TalkBack

### Energy Level Semantics (for UX copy consistency)
| Value | Label | Meaning | Icon |
|---|---|---|---|
| `high_focus` | High Focus | Deep work, full concentration needed | 🔥 or ↑ |
| `low_focus` | Low Focus | Routine tasks, some attention needed | ⚡ or → |
| `no_brainer` | No-Brainer | Autopilot tasks, near-zero cognitive load | 🌿 or ○ |
| `null` | (untagged) | No energy preference set | (none) |

Use these labels consistently across all UI copy, tooltips, and filter chip labels.

### Filter Behavior Edge Cases
- If user has Today view with 5 tasks, applies "High Focus" filter, and only 2 tasks match: show those 2 (the "Show more (N)" control should reflect the filtered count, not total)
- If all tasks are filtered out: show empty state "No High Focus tasks today" (not the generic empty state)

### What This Story Does NOT Include
- Automatic energy level suggestion based on time of day or wearable data (brainstorm idea — future phase)
- Energy level in recurring task templates
- Energy level as a sorting dimension (tasks sort by today_sort_order and due_date; energy is a filter, not a sorter, for MVP)
