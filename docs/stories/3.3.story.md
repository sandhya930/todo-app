# Story 3.3: Weekly Review Wizard

## Status: Draft

## Story

- As Alex (overwhelmed professional)
- I want a guided weekly review wizard that helps me process my Inbox, triage deferred tasks, and plan the week ahead
- so that my lists stay manageable and nothing important falls through the cracks

## Acceptance Criteria (ACs)

1. A weekly review prompt appears every Sunday between 6pm–9pm in the user's local timezone (default; day and time window configurable in Settings)
2. The wizard has exactly 5 steps: (1) Clear Inbox, (2) Review deferred tasks, (3) Review Someday, (4) Plan next week, (5) Celebrate wins
3. Each step shows tasks one at a time; available actions per task are: Schedule (set a due date), Move to Someday, Delete (with confirm), Keep as-is (no change)
4. The user can exit the wizard at any step and resume it at the same step on their next visit within 24 hours; after 24 hours the wizard resets to step 1
5. Completing the full 5-step wizard (or completing all tasks in each step) unlocks the "Weekly Review" badge (consumed by EP-04 Story 4.2 streak system — stub if not yet available)
6. The final "Celebrate wins" step shows: total tasks completed this week, streak count, top project by completion, and a "Well done, [name]!" message
7. A "Skip this review" option is available at any step and dismisses the wizard for the current week without penalty
8. Weekly review completion rate is tracked as a local analytics event (no server call required)

## Tasks / Subtasks

- [ ] **Task 1 — Weekly review prompt and scheduling** (AC: 1, 7)
  - [ ] 1.1 Store `weekly_review_day` (0=Sun–6=Sat, default 0) and `weekly_review_window_start` / `_end` (time strings, default "18:00"/"21:00") on User settings
  - [ ] 1.2 On app foreground: check if current time is within the configured review window and `last_weekly_review_completed_at` < start of this week
  - [ ] 1.3 If check passes and no review is in progress: show a dismissible banner in Today view: "🗓 Time for your weekly review — [Start] [Later]"
  - [ ] 1.4 "Later" dismisses the banner for 2 hours (stores `review_snoozed_until` in local state)
  - [ ] 1.5 Settings page: "Weekly Review" section with day picker and time window picker
  - [ ] 1.6 Write unit tests: prompt shown when in window + no review this week; not shown when review already done this week; not shown during snooze period

- [ ] **Task 2 — Wizard state management** (AC: 4, 8)
  - [ ] 2.1 Persist wizard progress: `{ currentStep: 1–5, stepItemIndex: number, startedAt: timestamp }` to local storage (survives app close)
  - [ ] 2.2 On wizard open: if saved progress exists and `startedAt > now() - 24h`, resume at saved step/index; else start fresh from step 1
  - [ ] 2.3 On exit ("Save & Exit" or device back): save current step and index before dismissing
  - [ ] 2.4 On "Skip this review": clear wizard state, set `last_weekly_review_completed_at = now()` (marks week as reviewed; no badge earned), log `{ event: 'weekly_review_skipped', step: currentStep }`
  - [ ] 2.5 Log analytics event on completion: `{ event: 'weekly_review_completed', durationMs, tasksScheduled, tasksMovedToSomeday, tasksDeleted }`
  - [ ] 2.6 Write unit tests: resume from step 3 after re-open; reset after 24h; skip clears state and marks week complete

- [ ] **Task 3 — Step 1: Clear Inbox** (AC: 2, 3)
  - [ ] 3.1 Show all tasks with `status = 'inbox'` one at a time; progress indicator "Step 1 of 5 — Inbox (N tasks)"
  - [ ] 3.2 Per-task actions: Schedule (opens date/time picker → sets due_date, status='active'), Move to Someday (status='someday'), Delete (confirm dialog → status='archived'), Keep (no change; moves to next task)
  - [ ] 3.3 "Keep" does not change the task but advances the wizard index; all Inbox tasks shown, not just a subset
  - [ ] 3.4 When all Inbox tasks processed (or Inbox is empty): auto-advance to step 2 with a brief "Inbox clear ✓" confirmation moment (1 second)
  - [ ] 3.5 Write component tests: each action updates task correctly; all tasks processed → auto-advance; empty inbox → immediate advance

- [ ] **Task 4 — Step 2: Review deferred tasks** (AC: 2, 3)
  - [ ] 4.1 Show all tasks with `status = 'deferred'` one at a time; progress indicator "Step 2 of 5 — Carried Over (N tasks)"
  - [ ] 4.2 Same four actions as Step 1 (Schedule / Move to Someday / Delete / Keep)
  - [ ] 4.3 Display `deferred_count` on each task card: "Carried over N times" (neutral tone)
  - [ ] 4.4 Auto-advance to step 3 when all deferred tasks processed
  - [ ] 4.5 Write component tests: deferred_count shown; actions identical to step 1; auto-advance

- [ ] **Task 5 — Step 3: Review Someday** (AC: 2, 3)
  - [ ] 5.1 Show all `status = 'someday'` tasks one at a time; progress indicator "Step 3 of 5 — Someday (N tasks)"
  - [ ] 5.2 Actions: Schedule (promotes to active with due date), Activate (status='inbox', no date), Delete, Keep
  - [ ] 5.3 Surfacing Someday items this way satisfies the requirement in EP-02 Story 2.5 AC 5
  - [ ] 5.4 Auto-advance to step 4 when all items processed; empty Someday → immediate advance
  - [ ] 5.5 Write component tests: "Activate" action sets status=inbox; "Schedule" sets active with date

- [ ] **Task 6 — Step 4: Plan next week** (AC: 2)
  - [ ] 6.1 Show tasks due in the next 7 days (status='active', due_date within next 7 days) grouped by day
  - [ ] 6.2 User can: reschedule any task (tap to change date), reorder within a day (drag), or leave as-is
  - [ ] 6.3 Quick-add: "Add a task for next week" input at the bottom; creates task with a date in the next 7 days
  - [ ] 6.4 "Done planning" button (no auto-advance in this step — user explicitly confirms readiness)
  - [ ] 6.5 Write component tests: upcoming tasks grouped by day, reschedule works, quick-add creates task in correct date

- [ ] **Task 7 — Step 5: Celebrate wins** (AC: 2, 5, 6)
  - [ ] 7.1 Display: tasks completed this week (count + list of titles, scrollable), current streak, top project by completed-task count this week, "Well done, [user_name]!" heading
  - [ ] 7.2 "Finish review" button: sets `last_weekly_review_completed_at = now()`, clears wizard state, dispatches `WEEKLY_REVIEW_COMPLETED` event (consumed by EP-04 streak/badge system)
  - [ ] 7.3 If EP-04 badge system not yet implemented: stub event dispatch as a no-op; badge shown as greyed-out "Coming soon" placeholder
  - [ ] 7.4 Write component tests: correct stats displayed; "Finish review" fires event and clears state

## Dev Notes

### Dependencies
- Task model with `status` values including `'inbox'`, `'deferred'`, `'someday'` (EP-01 1.1, EP-02 2.5, EP-03 3.2)
- `deferred_count` field (Story 3.2)
- `last_interacted_at` on Someday tasks (EP-02 Story 2.5) — interacting with a Someday task in the wizard should update this field
- EP-04 Story 4.2 (streak system) consumes the `WEEKLY_REVIEW_COMPLETED` event
- Settings page (needs to exist or be created alongside this story)

### Wizard Navigation Model
```
[Banner] → [Step 1: Inbox] → [Step 2: Deferred] → [Step 3: Someday]
        → [Step 4: Plan Next Week] → [Step 5: Celebrate] → [Done]
         ↑____________________ Resume from any step ___________________↑
```
- Progress bar at top: 5 segments, current step highlighted
- "Exit & save progress" button always visible (top-right)
- "Skip this review" link always visible (bottom, subtle)

### Empty Step Handling
If a step has 0 tasks (e.g., Inbox is already empty):
- Show step briefly with "All clear! Nothing here to review." + auto-advance after 1.5 seconds
- Do not skip the step silently — user should feel the satisfaction of each step completing

### Stats for Celebrate Step (AC 6)
- "Tasks completed this week" = tasks where `completed_at >= startOfWeek(now(), user.timezone)`
- "Top project" = project with most completed tasks this week (if tie: alphabetical)
- If streak system (EP-04) not yet done: show streak as 0 with a note "Streak tracking coming soon"

### Timezone
- All date comparisons use user's stored `timezone` preference (from deferral story 3.2)
- "This week" = Monday 00:00 to Sunday 23:59 in user's timezone

### What This Story Does NOT Include
- AI-assisted planning suggestions during step 4 (future enhancement)
- Sharing the weekly review summary externally (EP-04 Story 4.3 digest email is separate)
- Review for more than 7 days back
- Team-level or shared-list review

5. Complete all 5 steps → verify Step 5 shows completed tasks count and streak

7. Reopen same day → verify banner does not reappear (review already done this week)
