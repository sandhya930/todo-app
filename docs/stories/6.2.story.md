# Story 6.2: Markdown in Task Notes

## Status: Draft

## Story

- As Dev (power user / developer)
- I want Markdown formatting supported in task notes/descriptions
- so that I can add structured information, code snippets, and clickable links to tasks without fighting a rich-text editor

## Acceptance Criteria (ACs)

1. Task notes support a defined CommonMark subset (see Dev Notes); Markdown is rendered in view mode and editable as raw text in edit mode
2. An edit/preview toggle switches between raw Markdown (edit) and rendered output (view); no live split-pane — single pane only
3. Supported syntax: `**bold**`, `_italic_`, `# H1`, `## H2`, `### H3`, `- unordered list`, `1. ordered list`, `` `inline code` ``, triple-backtick code blocks with language hint, `[text](url)` links, `> blockquote`, `---` horizontal rule
4. Code blocks render with syntax highlighting for: JavaScript, TypeScript, Python, Bash, SQL, JSON
5. Images (`![alt](url)`) are NOT rendered — displayed as a plain `[image: alt text]` label; no remote image fetching
6. All Markdown content is sanitised server-side before storage and at render time to prevent XSS; no raw HTML allowed in input
7. Markdown in task notes is fully searchable (plain-text indexed, not rendered HTML)
8. The edit/preview toggle and rendered output replace the plain-text notes stub in Focus Mode (Story 2.2) and task detail view

## Tasks / Subtasks

- [ ] **Task 1 — Markdown rendering library integration** (AC: 1, 3, 4, 5)
  - [ ] 1.1 Select and integrate a Markdown rendering library:
    - Web: `marked` + `highlight.js` (lightweight, tree-shakeable); or `markdown-it` + `markdown-it-highlightjs`
    - React Native: `react-native-markdown-display` (renders to native components, not WebView)
    - Document choice in Change Log
  - [ ] 1.2 Configure renderer to the supported subset only (AC 3): disable HTML passthrough (`html: false`), disable raw HTML injection
  - [ ] 1.3 Image handling (AC 5): intercept `![alt](url)` tokens; render as `<span class="image-placeholder">[image: alt]</span>` — do not fetch or embed images
  - [ ] 1.4 Syntax highlighting: register only the 6 required languages (JS, TS, Python, Bash, SQL, JSON) to keep bundle size small; highlight.js supports tree-shaking
  - [ ] 1.5 Write unit tests: bold/italic/headings render correctly; code block with JS highlights; image token replaced with placeholder; unsupported HTML tag stripped

- [ ] **Task 2 — XSS sanitisation** (AC: 6)
  - [ ] 2.1 Server-side: sanitise `notes` field with `DOMPurify` (Node.js via `isomorphic-dompurify`) or `sanitize-html` before writing to DB; strip all HTML tags; allow only the Markdown subset (stored as plain Markdown text, not HTML)
  - [ ] 2.2 Render-time (client): after converting Markdown → HTML, pass output through `DOMPurify.sanitize()` before injecting into the DOM
  - [ ] 2.3 CSP header: ensure `Content-Security-Policy` includes `script-src 'self'` to prevent injected script execution even if sanitisation has edge cases
  - [ ] 2.4 Test with OWASP XSS Cheat Sheet payloads: verify all are neutralised both at storage and render time
  - [ ] 2.5 Write unit tests: `<script>alert(1)</script>` in notes → stripped at save and at render; `javascript:` URL in link → replaced with `#` or stripped; `onload=` attribute → stripped

- [ ] **Task 3 — Edit / preview toggle in task detail** (AC: 1, 2, 8)
  - [ ] 3.1 Task detail notes section: two modes controlled by a toggle button (pencil icon = edit, eye icon = preview)
  - [ ] 3.2 Edit mode: plain `<textarea>` (web) / `<TextInput multiline>` (RN) containing raw Markdown; monospace font; line numbers optional
  - [ ] 3.3 Preview mode: rendered Markdown output; read-only; links are tappable/clickable; code blocks show copy button (copies code to clipboard)
  - [ ] 3.4 Default mode: preview if notes are non-empty; edit if notes are empty (user is about to add notes for the first time)
  - [ ] 3.5 Auto-save: in edit mode, save notes to local DB on 1-second debounce after last keystroke (no explicit save button for notes)
  - [ ] 3.6 Replace plain-text stub in Story 2.2 Focus Mode: Focus Mode shows rendered preview of notes (read-only); no edit toggle in Focus Mode
  - [ ] 3.7 Write component tests: toggle switches mode; auto-save fires after debounce; default mode selection; Focus Mode shows preview only; copy button on code block copies correctly

- [ ] **Task 4 — Full-text search integration** (AC: 7)
  - [ ] 4.1 Markdown notes are stored as raw Markdown text; the search index uses the **plain-text representation** (strip Markdown syntax tokens before indexing)
  - [ ] 4.2 Create `stripMarkdown(text: string): string` utility: removes `**`, `_`, `#`, `` ` ``, `>`, `[text](url)` → `text`, `---`, list markers; returns readable plain text
  - [ ] 4.3 When a task's notes are saved/updated, re-index the stripped plain text in the search index
  - [ ] 4.4 Write unit tests: `stripMarkdown("**bold** and _italic_")` → `"bold and italic"`; link → text only; code block → code content preserved; headings → text only

- [ ] **Task 5 — Syntax highlighting theme** (AC: 4)
  - [ ] 5.1 Apply a syntax highlighting theme consistent with the app's design system: use a light theme in light mode, dark theme in dark mode (if dark mode is implemented; fall back to a single neutral theme if not)
  - [ ] 5.2 Language auto-detection fallback: if no language hint is provided in the code fence, render as plain code (no highlighting); do not attempt auto-detection (performance + accuracy tradeoff)
  - [ ] 5.3 Code block copy button: small "Copy" button top-right of each code block; copies raw code to clipboard; shows "Copied!" for 1.5 seconds
  - [ ] 5.4 Write visual/snapshot tests: JS code block renders with expected highlight classes; copy button present; "Copied!" state transitions

## Dev Notes

### Supported Markdown Subset (AC 3 — Exact Spec)

| Syntax | Example | Rendered as |
|---|---|---|
| Bold | `**text**` | `<strong>text</strong>` |
| Italic | `_text_` or `*text*` | `<em>text</em>` |
| Heading H1 | `# Heading` | `<h1>` |
| Heading H2 | `## Heading` | `<h2>` |
| Heading H3 | `### Heading` | `<h3>` |
| Unordered list | `- item` | `<ul><li>` |
| Ordered list | `1. item` | `<ol><li>` |
| Inline code | `` `code` `` | `<code>` |
| Code block | ` ```js\ncode\n``` ` | `<pre><code class="language-js">` |
| Link | `[text](url)` | `<a href="url">text</a>` |
| Blockquote | `> text` | `<blockquote>` |
| Horizontal rule | `---` | `<hr>` |
| Image | `![alt](url)` | `<span>[image: alt]</span>` |
| Raw HTML | `<div>` | Stripped entirely |
| Strikethrough | `~~text~~` | NOT supported (GFM extension — excluded) |
| Tables | `\| a \| b \|` | NOT supported (GFM extension — excluded) |
| Task checkboxes | `- [ ] item` | NOT supported (conflicts with SubTask feature) |

### XSS Attack Vectors to Test (AC 6)
- `<script>alert(1)</script>` — direct script injection
- `<img src=x onerror=alert(1)>` — event handler on image
- `[click me](javascript:alert(1))` — javascript: URL in link
- `<a href="data:text/html,<script>alert(1)</script>">` — data URL
- ` ```html\n<script>alert(1)</script>\n``` ` — script in code block (must render as literal text, not execute)

All five must be neutralised at both storage time AND render time.

### Bundle Size Concern
- `highlight.js` full bundle is ~800KB; tree-shaking to 6 languages should be ~80KB
- `marked` is ~45KB; `markdown-it` is ~60KB — either is acceptable
- `DOMPurify` is ~45KB
- Total Markdown feature addition to web bundle: target ≤ 200KB gzipped

### React Native Markdown
- `react-native-markdown-display` renders Markdown to native RN components (no WebView)
- Syntax highlighting in RN: use a separate `SyntaxHighlighter` component that accepts raw code + language; renders as styled `<Text>` spans
- Copy button in RN: use `Clipboard.setString()` via `@react-native-clipboard/clipboard`

### Notes Auto-save (Task 3.5)
- Debounce 1 second from last keystroke — matches typical "fast typer" experience
- Save to local DB immediately (offline-first); CRDT sync picks it up asynchronously
- No "unsaved changes" indicator needed — auto-save is transparent
- If app is killed mid-debounce: last saved state is used (may lose up to 1 second of typing — acceptable)

### What This Story Does NOT Include
- Full GFM (GitHub-Flavoured Markdown) — tables, strikethrough, task checkboxes excluded
- WYSIWYG editor (no toolbar, no button-based formatting — raw Markdown only)
- Markdown in task titles (titles are plain text only)
- Markdown in sub-task titles (plain text only)
- Image upload/attachment (images stripped — no storage feature)
