# Story 5.2: Shared Lists

## Status: Draft

## Story

- As a family manager / household user
- I want to share a task list (project) with other people and assign tasks to individuals
- so that household responsibilities are transparent and everyone knows exactly what they own

## Acceptance Criteria (ACs)

1. Any project can be shared via invite link or email to up to 10 members (free tier); Pro tier: unlimited members
2. Tasks within a shared list can be assigned to any member; the assignee receives a push notification and email notification
3. All members see the full shared list and task status in real time (sync propagation ≤ 3 seconds between members)
4. Only the task's assignee or the list owner can mark a task as complete
5. The list owner can set per-member permissions: View Only / Can Edit / Can Assign
6. Leaving a shared list removes all its tasks from the leaving user's views; the tasks and list remain intact for other members
7. Shared lists are visually distinguished from personal lists: a people icon badge on the project in the sidebar and on task cards from shared lists
8. Free tier: max 1 shared list; Pro tier: unlimited shared lists. Attempting to share a second list on free tier shows an upgrade prompt

- CRDT sync infrastructure (EP-01 1.1 offline-first) — shared lists use the same sync path; server must broadcast to multiple users

### CRDT Multi-User Sync Extension
Personal tasks sync between one user's devices. Shared lists require the sync server to broadcast changes to **multiple users' devices**:
- Each `ProjectMember` (active) should subscribe to the project's CRDT document
- When any member writes a change, the server fans out the CRDT update to all other active members' connected sessions
- This is a significant infrastructure extension — flag for architecture review before implementation

### Permission Model Summary

| Action | View Only | Can Edit | Can Assign | Owner |
|---|---|---|---|---|
| View tasks | ✓ | ✓ | ✓ | ✓ |
| Create tasks | ✗ | ✓ | ✓ | ✓ |
| Edit tasks | ✗ | ✓ | ✓ | ✓ |
| Assign tasks | ✗ | ✗ | ✓ | ✓ |
| Complete own tasks | ✗ | ✓ | ✓ | ✓ |
| Complete any task | ✗ | ✗ | ✗ | ✓ |
| Change permissions | ✗ | ✗ | ✗ | ✓ |
| Remove members | ✗ | ✗ | ✗ | ✓ |

### `assignee_user_id` Field on Task
- Add `assignee_user_id` (nullable FK → User) to Task model
- Only meaningful for tasks in shared lists; personal tasks always have `assignee_user_id = null`
- This is separate from accountability partner (`accountability_partner_id` in Story 5.1) — assignee is the person responsible for completing; partner is just a cheerleader

### Free Tier Limits
- 1 shared list (free) vs unlimited (Pro)
- 10 members per shared list (free) — note: Pro also capped at 10 in epics spec; clarify with PO whether Pro lifts this cap too
- Both limits enforced server-side

### What This Story Does NOT Include
- Shared list notifications (someone adds a task to your shared list — notification) — future story
- Comments on tasks in shared lists
- Transferring list ownership
- Shared lists visible to non-members (public lists)
- Role-based task templates for shared lists (e.g., chore rotation logic)
