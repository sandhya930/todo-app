# Story 4.1: Completion Animations

## Status: Draft

## Story

- As Jordan (ADHD user) and any user
- I want a satisfying animation when I complete a task
- so that finishing feels genuinely rewarding and I'm motivated to do it again

## Acceptance Criteria (ACs)

1. Completing a task triggers a particle/confetti animation originating from the completed task card; animation duration is ≤ 800ms
2. A subtle chime sound plays on completion by default; the sound respects the device's silent/mute switch and system volume
3. Animation style is configurable in Settings → Display → Completion Animation: On (full particles) / Subtle (fade only) / Off (no animation, no sound)
4. Completing the last task in the Today view triggers an enhanced "All done!" full-screen moment: larger animation, celebratory message, auto-dismisses after 2 seconds or on tap
5. All animations are GPU-accelerated; no frame-rate drops below 60fps on supported devices (iPhone 12+, Android mid-range 2021+)
6. The iOS/Android "Reduce Motion" accessibility setting disables particle animations and substitutes a simple opacity-fade (≤ 300ms); sound still plays unless muted
7. The animation system is implemented as a standalone `CompletionAnimationService` that other stories (Focus Mode 2.2, Sub-tasks 4.4) can call without duplicating logic

## Tasks / Subtasks

- [ ] **Task 1 — CompletionAnimationService** (AC: 1, 3, 6, 7)
  - [ ] 1.1 Create `CompletionAnimationService` with a single public method: `play(origin: ViewRef | Position, type: 'task' | 'subtask' | 'all-done')`
    - `type = 'task'`: standard confetti burst from task card position
    - `type = 'subtask'`: smaller, lighter animation (used by Story 4.4)
    - `type = 'all-done'`: full-screen celebration (used by AC 4)
  - [ ] 1.2 Check `Reduce Motion` setting (platform API: `AccessibilityInfo.isReduceMotionEnabled` on RN, `prefers-reduced-motion` media query on web) on each call; swap to fade if enabled
  - [ ] 1.3 Check user's animation preference (from Settings store): On / Subtle / Off; skip animation entirely if Off; use fade for Subtle
  - [ ] 1.4 Service is importable by: task list components, Focus Mode (Story 2.2), sub-task completion (Story 4.4)
  - [ ] 1.5 Write unit tests: correct animation type dispatched per `type` param; Reduce Motion → fade substituted; Off → no animation dispatched; Subtle → fade only

- [ ] **Task 2 — Particle/confetti animation implementation** (AC: 1, 5)
  - [ ] 2.1 Implement confetti burst using a GPU-accelerated approach:
    - React Native: `react-native-reanimated` + `react-native-confetti-cannon` or equivalent; run on the UI thread (not JS thread)
    - Web: CSS `@keyframes` + `transform`/`opacity` only (no `top`/`left` for GPU compositing); or a lightweight canvas-based library (e.g., `canvas-confetti`)
  - [ ] 2.2 Particle count: 30–50 particles; colors derived from the task's project color (falls back to brand palette if no project)
  - [ ] 2.3 Duration: 800ms total; particles fade out naturally; no lingering elements after animation ends
  - [ ] 2.4 Performance test on target devices: confirm 60fps throughout animation; profile on Android mid-range if possible
  - [ ] 2.5 Write snapshot/visual test: animation component renders without errors; particles removed from DOM/view tree after 800ms

- [ ] **Task 3 — Completion sound** (AC: 2)
  - [ ] 3.1 Bundle a single short chime sound file (≤ 50KB, MP3/AAC, ~200ms duration)
  - [ ] 3.2 Play sound via platform audio API respecting system silent switch:
    - iOS: use `AVAudioSession` category `Ambient` (respects silent switch)
    - Android: `AudioManager.STREAM_NOTIFICATION` (respects system volume, not affected by Do Not Disturb at this stream level)
    - Web: `AudioContext` with `gainNode`; check `document.hidden` — do not play if tab is backgrounded
  - [ ] 3.3 Sound plays concurrently with animation, not after
  - [ ] 3.4 Write unit tests: sound plays when setting is On; sound skipped when setting is Off; sound skipped when Subtle is selected (visual-only mode)

- [ ] **Task 4 — "All done!" full-screen moment** (AC: 4)
  - [ ] 4.1 After completing the last active task in Today view, detect `todayActiveTasks.length === 0` post-completion
  - [ ] 4.2 Render a full-screen overlay: large confetti burst (3× particle count), message "All done! 🎉", sub-message "You crushed it today.", auto-dismiss timer (2 seconds)
  - [ ] 4.3 Tap anywhere on overlay → immediate dismiss
  - [ ] 4.4 "All done!" overlay does NOT appear if user is in Focus Mode (Focus Mode has its own completion flow — Story 2.2)
  - [ ] 4.5 Write component tests: overlay renders when last task completed; auto-dismisses after 2s; tap dismisses immediately; not shown in Focus Mode

- [ ] **Task 5 — Settings: animation preference** (AC: 3)
  - [ ] 5.1 Settings → Display → "Completion Animation" section: three radio options: "Full (confetti + sound)", "Subtle (fade only)", "Off"
  - [ ] 5.2 Preference stored in user settings (local + synced); default: "Full"
  - [ ] 5.3 Live preview: tapping each option triggers a sample animation so user can see the effect without completing a real task
  - [ ] 5.4 Write component tests: preference persists after app restart; live preview fires correct animation type

- [ ] **Task 6 — Wire into task completion flow** (AC: 1, 7)
  - [ ] 6.1 In `completeTask()` service or task card component: after marking task complete, call `CompletionAnimationService.play(taskCardRef, 'task')`
  - [ ] 6.2 In Focus Mode (Story 2.2): replace the stub animation with a call to `CompletionAnimationService.play(null, 'task')` (no origin position needed in full-screen mode)
  - [ ] 6.3 Write integration test: `completeTask()` → animation service called with correct type; Focus Mode completion triggers animation

## Dev Notes

### Dependencies
- `completeTask()` service (EP-01 Story 1.1)
- Today view task list (EP-02 Story 2.1) — "all done" detection happens here
- Focus Mode (EP-02 Story 2.2) — replace stub with real service call
- Sub-tasks (Story 4.4) — will call `CompletionAnimationService.play(ref, 'subtask')`

### GPU Compositing Rules (AC 5)
Only use CSS properties that trigger GPU compositing to avoid layout/paint:
- **Allowed:** `transform` (translate, scale, rotate), `opacity`
- **Not allowed:** `top`, `left`, `width`, `height`, `margin` during animation
On React Native: use `react-native-reanimated` worklets running on the UI thread, not `Animated` API (which runs on JS thread)

### Sound File Spec
- Format: MP3 + AAC fallback
- Duration: ~200ms (short, punchy — not a jingle)
- Volume: normalised to -12 LUFS (not jarring at full system volume)
- Licence: royalty-free or custom; do not use copyrighted material

### Reduce Motion (AC 6)
- Check on every `play()` call, not just on app launch — user may toggle it during a session
- `AccessibilityInfo.addEventListener('reduceMotionChanged', handler)` on React Native
- Web: `window.matchMedia('(prefers-reduced-motion: reduce)').addEventListener('change', handler)`

### What This Story Does NOT Include
- Haptic feedback (future enhancement; requires device capability check)
- Custom animation themes / skins
- Animations on task deletion or deferral
- Streak milestone animations (handled in Story 4.2)
