# Story 2.2: Focus Mode

## Status: Draft

## Story

- As Jordan (ADHD user)
- I want to enter Focus Mode to see only one task at a time, full-screen
- so that I can initiate work without being overwhelmed by the full list

## Acceptance Criteria (ACs)

1. Focus Mode is accessible via a button on any task card and via keyboard shortcut `F` (web/desktop) or `⌘⇧F`
2. Focus Mode is a full-screen view showing only: task title, notes (if any), sub-tasks checklist (if any — stub if EP-04 Story 4.4 not yet done), a timer button, and a complete button
3. All other tasks and navigation chrome are hidden while Focus Mode is active
4. Escape key (web) or swipe-down gesture (mobile) exits Focus Mode and returns the user to the exact view they came from
5. Completing the task in Focus Mode triggers the completion animation (stub a simple fade if EP-04 Story 4.1 not yet done), then surfaces a "Next up" prompt: shows the title of the next recommended task with "Start" and "Exit" buttons
6. Focus Mode state is preserved if the app is backgrounded and reopened — same task is shown on return
7. Focus Mode is accessible: all interactive elements reachable by keyboard (web); VoiceOver / TalkBack announces the task title and action buttons on entry

  - [ ] 2.4 Write tests: Focus button on card triggers FocusModeScreen with correct task; keyboard shortcut fires correctly

  - [ ] 3.1 Escape key (web) dismisses Focus Mode; swipe-down gesture (mobile) dismisses
  - [ ] 3.2 Track the originating view/route before entering Focus Mode; on exit navigate back to it (not always to Today)
  - [ ] 3.3 App backgrounded while in Focus Mode: on foreground restore, return to Focus Mode with same task (read `activeFocusTaskId` from persisted state)
  - [ ] 3.4 Write tests: exit returns to Inbox (if entered from Inbox), to Today (if entered from Today); background/foreground restores correctly

- [ ] **Task 4 — Completion flow in Focus Mode** (AC: 5)
  - [ ] 4.1 "Mark complete" button calls `completeTask(taskId)` (same service used in all views)
  - [ ] 4.2 Completion animation plays (use EP-04 stub or completed animation if available)
  - [ ] 4.3 After animation: show "Next up" card — display title of the next recommended Today task (use simple priority: next task by `today_sort_order`; no AI needed here)
  - [ ] 4.4 "Start" button on "Next up" card replaces current Focus Mode content with next task (no exit/re-entry)
  - [ ] 4.5 "Exit" button on "Next up" card dismisses Focus Mode and returns to originating view
  - [ ] 4.6 If no next task available: show "You're all done! 🎉" message with only an "Exit" button
  - [ ] 4.7 Write tests: complete task → animation stub → next-up shown; "Start" transitions to next task; "Exit" navigates back; all done state shown when last task completed

- [ ] **Task 5 — Timer button (stub)** (AC: 2)

  - [ ] 5.2 Default timer duration: 25 minutes (Pomodoro); user can adjust in the timer UI
  - [ ] 5.3 Timer runs in the foreground only (no background notification for this story — that is EP-03 scope)

## Dev Notes

- Task model and `completeTask()` service must exist (EP-01 Story 1.1)
- Today view task list needed for "Next up" lookup (EP-02 Story 2.1)
- Focus Mode is a self-contained overlay — it does not depend on Projects (2.3) or Energy Tags (2.4)

### Stub Plan for Unimplemented Dependencies

| Sub-tasks (EP-04 Story 4.4) | Show empty checklist section | Replace when 4.4 merges |
| Markdown notes (EP-06 Story 6.2) | Render notes as plain text | Replace when 6.2 merges |

### Emotional Design (from PRD Persona Jordan)
- Focus Mode is the primary ADHD-supportive feature — its design is critical
- The full-screen takeover removes visual overwhelm; the single task + timer creates a "start ritual"
- Exit must be frictionless (swipe-down, not a multi-step modal) — avoid trapping the user
- "Next up" should feel like an invitation, not a demand — copy must be warm

### Keyboard Accessibility (AC 7, from PRD §7.4)
- Tab order in Focus Mode: title (read-only) → notes (if present) → sub-tasks (if present) → timer button → complete button → exit button
- No focus trap: pressing Tab past the last element should NOT cycle back (user may want to exit via keyboard)
- Escape key must always exit Focus Mode regardless of focus position

### State Persistence (AC 6)
- Store `activeFocusTaskId` in persistent app state (AsyncStorage / localStorage)
- Clear it on: task completion, "Exit" button, or task deletion
- On app launch: if `activeFocusTaskId` is set and the task still exists (not completed/deleted), offer to resume: "Resume focus on [task]?" with Yes/No

### What This Story Does NOT Include
- Background timer with push notification ("Pomodoro complete")
- Multiple concurrent focus sessions
- Focus Mode for sub-tasks individually
- Analytics on Focus Mode usage (tracked in a future instrumentation story)
