# Story 4.4: Sub-Tasks / Micro-Completions

## Status: Draft

## Story

- As Jordan (ADHD user)
- I want to break a task into small sub-steps I can check off one by one
- so that I can start and make visible progress even when I can't complete the whole thing at once

## Acceptance Criteria (ACs)

1. Any task can have up to 20 sub-tasks; each sub-task has a title (required, max 200 chars) and a completion checkbox
2. Sub-tasks are added, edited, reordered, and deleted inline within the task detail view — no separate screen required
3. Completing a sub-task triggers a smaller celebration animation (lighter than full task completion) via `CompletionAnimationService`
4. A progress bar on the parent task card shows X/Y sub-tasks complete (e.g., "3/5")
5. When all sub-tasks are checked, the app prompts: "All steps done! Mark task complete?" with "Yes, complete it" and "Not yet" buttons; the parent task is NOT auto-completed without user confirmation
6. In Focus Mode (Story 2.2), sub-tasks are displayed as a numbered checklist; completing all triggers the full task completion prompt (AC 5)
7. Sub-tasks are included in full-text search results (matched sub-task title surfaces the parent task)
8. Sub-tasks sync across devices via the standard CRDT path; order is preserved
9. The sub-task stub in Story 2.2 (Focus Mode) and any other stories that stubbed sub-tasks are replaced by this implementation

## Tasks / Subtasks

- [ ] **Task 1 — Sub-task data model** (AC: 1, 8)
  - [ ] 1.1 Define `SubTask` schema: `id` (UUID), `task_id` (FK → Task), `title` (string, max 200 chars), `completed` (boolean, default false), `completed_at` (nullable timestamp), `sort_order` (integer), `created_at`
  - [ ] 1.2 Sub-tasks are child records of a Task; cascade delete when parent Task is deleted/archived
  - [ ] 1.3 Enforce max 20 sub-tasks per task at the service layer (not just UI): `createSubTask()` returns error if count ≥ 20
  - [ ] 1.4 Sub-tasks participate in CRDT sync via the same sync path as Tasks (sub-tasks are independent sync objects, not embedded JSON — allows individual sub-task updates without full task re-sync)
  - [ ] 1.5 Write unit tests: create sub-task, max-20 enforcement, cascade delete, sort_order assignment (auto-increment)

- [ ] **Task 2 — Sub-task management UI in task detail** (AC: 2)
  - [ ] 2.1 Task detail view: "Add a step…" inline input at the bottom of the sub-task list; pressing Enter saves and adds another input; pressing Escape cancels
  - [ ] 2.2 Each sub-task row: checkbox (left), title (editable inline on tap/click), drag handle (right, for reordering), delete button (× on hover/long-press)
  - [ ] 2.3 Drag-and-drop reorder: updates `sort_order` for affected sub-tasks; same drag component as Today view
  - [ ] 2.4 Max 20 reached: "Add a step…" input hidden and replaced with "Maximum 20 steps reached" message
  - [ ] 2.5 Write component tests: add sub-task, edit inline, delete, reorder changes sort_order, max-20 hides input

- [ ] **Task 3 — Sub-task completion + micro-animation** (AC: 3, 4)
  - [ ] 3.1 Tapping checkbox: sets `completed = true`, `completed_at = now()`, triggers `CompletionAnimationService.play(checkboxRef, 'subtask')`
  - [ ] 3.2 `'subtask'` animation type (defined in Story 4.1): smaller particle burst, no sound (to avoid audio fatigue on rapid completions)
  - [ ] 3.3 Progress bar on parent task card: `{completedCount}/{totalCount}`; bar fill = `completedCount / totalCount`; shown only when sub-tasks exist; hidden when task has 0 sub-tasks
  - [ ] 3.4 Progress bar visible in: Today view task cards, Inbox task cards, Project view task cards
  - [ ] 3.5 Write component tests: progress bar shows correct fraction; bar hidden when no sub-tasks; animation fired on check; animation NOT fired on uncheck (undoing completion)

- [ ] **Task 4 — All-done prompt and parent completion** (AC: 5)
  - [ ] 4.1 After completing the last uncompleted sub-task: check if `completedCount === totalCount`
  - [ ] 4.2 If all complete: show a non-blocking bottom sheet or dialog: "All steps done! Ready to mark [task title] complete?" with "Yes, complete it" (calls `completeTask()`) and "Not yet" (dismisses, task remains active)
  - [ ] 4.3 "Yes, complete it" triggers the full task completion animation (Story 4.1 `'task'` type)
  - [ ] 4.4 "Not yet": task remains with all sub-tasks checked; user can uncheck sub-tasks at any time
  - [ ] 4.5 Prompt is NOT shown if the parent task is completed directly (without sub-tasks being individually checked)
  - [ ] 4.6 Write component tests: prompt shown when last sub-task checked; "Yes" calls completeTask; "Not yet" dismisses; no prompt when completing parent directly

- [ ] **Task 5 — Focus Mode integration** (AC: 6, 9)
  - [ ] 5.1 Replace the sub-task stub in Story 2.2 Focus Mode with the real `SubTask` list component
  - [ ] 5.2 Focus Mode sub-task display: numbered checklist (not draggable in Focus Mode — simplified view); each item shows title + checkbox; completing items triggers micro-animation
  - [ ] 5.3 When all sub-tasks completed in Focus Mode: show the all-done prompt (AC 5) directly within the Focus Mode view (no separate sheet — inline within the focus overlay)
  - [ ] 5.4 Write component tests: sub-tasks render in Focus Mode; completing all triggers prompt; "Yes" transitions to next-task flow (Story 2.2 Task 4)

- [ ] **Task 6 — Search integration** (AC: 7)
  - [ ] 6.1 Sub-task titles are indexed in the full-text search
  - [ ] 6.2 Search result for a sub-task match: surfaces the parent task card with the matching sub-task title shown as a subtitle ("Step: [sub-task title]")
  - [ ] 6.3 Write integration test: search for a sub-task title → parent task returned with sub-task highlighted in result

## Dev Notes

### Dependencies
- Task model (EP-01 Story 1.1) — `SubTask` is a child of `Task`
- `CompletionAnimationService` (Story 4.1) — must be implemented before sub-task animation works; if not yet done, use the stub pattern (opacity fade)
- Focus Mode (EP-02 Story 2.2) — replace stub
- `completeTask()` service — called from "Yes, complete it" prompt
- Today/Inbox/Project task card components — add progress bar display

### CRDT Sync for Sub-Tasks (AC 8)
Sub-tasks as independent sync records (not embedded in task JSON) is deliberate:
- Allows two devices to complete different sub-tasks simultaneously without conflict
- Each `SubTask` has its own CRDT identity (id + fields)
- `sort_order` conflicts: last-write-wins is acceptable for sort_order (minor cosmetic)
- `completed` conflicts: last-write-wins (completing a sub-task on two devices simultaneously is fine)

### Sub-task Completion Does NOT Affect Streak Directly
- Only parent task completion via `completeTask()` calls `StreakService.recordCompletion()`
- Sub-task completion does NOT directly increment the streak or `DailyCompletion` count
- However, completing all sub-tasks → confirming parent task complete → `completeTask()` called → streak records normally

### Unchecking a Completed Sub-task
- Allow: user can uncheck a completed sub-task (sets `completed = false`, clears `completed_at`)
- No reverse animation — just a visual checkbox toggle
- If parent task was completed via the prompt and user unchecks a sub-task: the parent task remains completed (do not auto-reopen parent on sub-task uncheck)

### Progress Bar Design
- Show as `"3/5 ●●●○○"` or a thin bar fill — design decision; both acceptable
- Colour: brand primary or project colour
- Not shown on tasks with 0 sub-tasks (clean card for simple tasks)
- In Today view: progress bar appears as a subtle secondary line below the task title

### What This Story Does NOT Include
- Nested sub-tasks (sub-tasks of sub-tasks) — flat list only at MVP
- Sub-task due dates or energy levels
- Sub-task assignment to different users (collaboration scoped to parent task only)
- Sub-task templates or copying sub-tasks between tasks
