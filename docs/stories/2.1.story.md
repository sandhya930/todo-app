# Story 2.1: Today View

## Status: Draft

## Story

- As Alex (overwhelmed professional)
- I want a "Today" view that shows only my prioritized tasks for today
- so that I open the app and immediately know what to focus on without scrolling through a wall of tasks

## Acceptance Criteria (ACs)

1. Today view is the default home screen after onboarding completes
2. Shows up to 5 tasks by default; a "Show more (N)" control expands to reveal all today-assigned tasks
3. Tasks shown in Today: those with `due_date = today` (calendar date, user's local timezone) plus tasks manually pinned to today by the user
4. If Today is empty, a friendly empty state is shown (not a blank screen) with a "Plan your day" CTA that opens the Inbox
5. Tasks within Today are reorderable by drag-and-drop; order is persisted per-user
6. An "Add to today" action is available on any task from any view (Inbox, Upcoming, Projects, Someday)
7. Today view is fully functional offline; no tasks disappear or error due to connectivity loss
8. The view header shows today's date (e.g., "Sunday, March 15") and the user's current streak count (from EP-04 Story 4.2 — stub with 0 if not yet implemented)

## Dev Notes

### Dependencies on EP-01
- Task model (`id`, `title`, `status`, `due_date`, `created_at`) must exist (Story 1.1)
- This story extends the Task model with `pinned_today` and `today_sort_order` — coordinate with 1.1 if in parallel dev
- `createTask()` service from 1.1 is used for any tasks added from the Today empty state

### Streak Badge (AC 8)
- The streak counter is implemented in EP-04 Story 4.2
- For this story: render a streak badge component that accepts a `count` prop; default to 0 if EP-04 not yet merged
- Do not block Today view release on streak implementation

### Today View Task Scope (AC 3)
- "Due today" = tasks where `date(due_date) = date(now())` in user's **local timezone** — not UTC
- A task due at 11pm in New York should appear in the New York user's Today view, not the UTC-next-day view
- Use the device's local timezone for all date comparisons in the client

### "Show More" Behavior
- The default limit of 5 is a UX decision, not a data limit
- "Show more" reveals tasks in the same view (no new screen); collapsed state is NOT persisted — always shows 5 on next open
- The count in "Show more (N)" shows how many additional tasks exist beyond the visible 5

### Empty State Design Note
- The empty state is emotionally important (PRD §2.2: shame-avoidance)
- Copy must be encouraging, not neutral: "Nothing planned for today yet" + CTA, not "No tasks"
- Confirm copy with design/PO before implementation

### What This Story Does NOT Include
- AI prioritization of Today tasks (EP-03 Story 3.1 — "Pick for me")
- Deferral mechanic (EP-03 Story 3.2)
- Completion animations (EP-04 Story 4.1)
- The streak counter logic itself (EP-04 Story 4.2) — only the display stub
