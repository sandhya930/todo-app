## Technique 1 — First Principles Thinking

*Strip away assumptions. What is a to-do list at its most fundamental level?*

1. A to-do list is a **memory prosthetic** — it offloads cognitive load so the brain can focus.
2. The core unit is a **commitment**: something you intend to do in the future.
3. A task has three essential properties: **identity** (what), **state** (done/not-done), and optionally **time** (when).
4. Everything else (tags, priorities, reminders, collaborators) is scaffolding on top of those three properties.
5. The real problem being solved is not list-making — it is **reducing the gap between intention and action**.
6. Current apps over-optimize for capture and under-optimize for completion.
7. A list that never gets done is worse than no list — it creates guilt and anxiety.
8. Re-framing: the app's KPI should be **tasks completed per week**, not tasks created.

## Technique 2 — SCAMPER

### Substitute
9. Substitute text tasks with **voice memos** automatically transcribed and parsed into structured tasks.
10. Substitute manual due dates with **AI-inferred deadlines** based on calendar context and past behavior.
11. Substitute checkboxes with **sliding completion percentages** (0%→100%) for nuanced progress tracking.
12. Substitute priority labels with **energy-level tags** (High Focus / Low Focus / No-Brainer).
13. Substitute lists with a **Kanban-style swim lane** view that mirrors a personal workflow.

### Combine
14. Combine a to-do list with a **habit tracker** — recurring tasks build streaks and momentum.
15. Combine task capture with **note-taking** (each task can expand into a full document).
16. Combine to-do list with a **time-blocking calendar** that auto-schedules tasks into open slots.
17. Combine task management with a **journaling layer** — end-of-day reflection on completed/missed tasks.
18. Combine to-do + **email inbox** — turn emails into tasks with one click.

### Adapt
19. Adapt **GTD (Getting Things Done)** methodology into built-in workflow: capture → clarify → organize → review → engage.
20. Adapt the **Ivy Lee Method**: each night, set exactly 6 tasks for tomorrow in priority order.
21. Adapt **Scrum sprint planning** for personal use — weekly sprints with a backlog, active sprint, and retrospective.
22. Adapt **traffic light systems** (red/yellow/green) for task health indicators.

### Modify / Magnify
23. Magnify context: tasks carry **full context packages** — attachments, links, notes, related tasks.
24. Modify the completion gesture: instead of a tap, require a **swipe with confirmation note** to prevent accidental completions.
25. Magnify the social layer: share task lists with accountability partners who get notified on completions.
26. Magnify time awareness: show a **real-time countdown timer** on the most urgent task.

### Put to Other Uses
27. Use the task database as a **personal productivity analytics dashboard** — visualize patterns over weeks/months.
28. Use completed-task history to **generate weekly accomplishment reports** for performance reviews.
29. Use task metadata to **train a personal ML model** that predicts how long your tasks actually take.

### Eliminate
30. Eliminate the concept of overdue tasks — replace with **"deferred"** status that auto-reschedules without guilt.
31. Eliminate manual input entirely for recurring tasks — **smart detection** after 3 occurrences at the same time.
32. Eliminate sub-tasks — instead use **linked tasks** (parent/child graph, not hierarchy).

### Reverse
33. Reverse the flow: start from a **goal**, then auto-decompose into milestone tasks and daily actions.
34. Reverse notifications: instead of reminding you to start, remind you when you've been working on one thing too long.
35. Reverse ownership: the app assigns tasks to you based on your declared priorities and available time.

98. Top competitors: Todoist, Things 3, TickTick, OmniFocus, Microsoft To Do, Notion, Linear.
99. Average user retention in productivity apps drops 60% within the first month.

### Red Hat (Emotions & Intuition)
101. Users feel **guilt** when looking at overdue tasks — the app should feel like a coach, not a judge.
102. Completion animations trigger **dopamine** — invest in delightful micro-interactions.
103. Users feel **overwhelmed** when lists grow too long — progressive disclosure is emotionally necessary.
104. There is a **pride factor** — users want to show off their productivity to others.

### Yellow Hat (Benefits & Optimism)
105. A well-designed app can meaningfully reduce cognitive load and stress.
106. Habit-forming loop (capture → plan → do → reflect) creates high long-term retention.
107. AI-assisted scheduling is a genuine time-saver that most apps have not fully solved yet.
108. There is a huge underserved market in **neurodivergent users** (ADHD, autism).

### Black Hat (Risks & Caution)
109. Feature bloat is the #1 cause of churn — resist adding complexity.
110. Privacy concerns with AI-read task content — need transparent data policies.
111. Notification fatigue causes users to mute the app and eventually abandon it.
112. Competing with free apps (Microsoft To Do, Apple Reminders) requires clear premium value.

### Green Hat (Creativity)

## Consolidated Feature Concepts (Prioritized)

### Core (MVP)
- Instant task capture (text, voice, email)
- Natural language date parsing ("next Tuesday", "in 3 days")
- Smart lists: Today, Upcoming, Someday, Completed
- Satisfying completion animations
- Offline-first with real-time sync
- Full-text search

### High-Value Differentiators
- **One task at a time** focus mode
- **AI daily planner**: surfaces the optimal task given time, energy, and deadlines
- **Goal → task decomposition**: break goals into actionable steps automatically
- **Accountability partner** sharing with completion notifications
- **Productivity analytics**: trends, streaks, completion rates

### Power User Features
- Full keyboard shortcuts & Vim navigation
- Markdown in task descriptions
- Open REST API + webhooks
- CLI companion
- Automation rules ("when task X completes, create task Y")

### Delight & Retention Features
- **Streak system** with milestone rewards
- **Weekly accomplishment digest** email
- **Reusable checklist templates**
- **"Pick for me"** AI button
- **Task time estimator** (learns from your actual history)

## Emerging Themes

1. **Frictionless capture is table stakes** — if it's not instant, users abandon the app.
2. **The guilt problem** — apps punish non-completion; the winner will celebrate and forgive instead.
3. **Focus over features** — the less you show, the more gets done.
4. **AI as a planning partner**, not just a search shortcut.
5. **Neurodivergent inclusion** is an underserved, vocal, loyal market segment.
6. **Social accountability** outperforms all notification strategies for follow-through.
7. **Personal analytics** transform a utility into an identity product.
